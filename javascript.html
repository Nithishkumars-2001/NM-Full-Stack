JavaScript

JavaScript brings web pages to life. This file covers the core language features and browser APIs every full stack developer relies on.

JavaScript is a lightweight, interpreted programming language originally built to make web pages interactive. It runs natively inside every modern web browser, and — thanks to runtimes like Node.js — can also run on servers, making it usable across the entire stack.

  • var — function-scoped, can be redeclared and updated; largely avoided in modern code
  • let — block-scoped, can be updated but not redeclared in the same scope
  • const — block-scoped, cannot be reassigned after declaration (though objects/arrays it holds can still be mutated)
let count = 1;
count = 2;        // OK

const name = "NM";
name = "X";        // Error

JavaScript has primitive types — String, Number, Boolean, null, undefined, Symbol, BigInt — and one reference type, Object (which includes arrays and functions). The typeof operator checks a value's type at runtime.

== is the loose equality operator: it converts both values to the same type before comparing. === is the strict equality operator: it compares both value and type without any conversion, and is almost always the safer choice.

'5' == 5    // true  (type coerced)
'5' === 5   // false (different types)

A function is a reusable block of code. Arrow functions are a shorter syntax introduced in ES6 that also handle the this keyword differently — they don't have their own this, instead inheriting it from the surrounding scope, which is especially useful inside callbacks. 

function add(a, b){ return a + b; }

const add2 = (a, b) => a + b;

The DOM (Document Object Model) is the browser's in-memory, tree-like representation of the HTML page. JavaScript can read and change it — adding, removing or updating elements — which is how a page updates without a full reload.

const el = document.querySelector('#title');
el.textContent = 'Updated!';
el.classList.add('highlight');

Events are actions the browser can detect — a click, a keypress, a form submit, a page load. An event listener is a function registered to run automatically when a specific event happens on a specific element.

document.querySelector('button')
  .addEventListener('click', () => {
    alert('Button clicked!');
  });

JavaScript is single-threaded, so long-running tasks like network requests are handled asynchronously so they don't block the rest of the page.

  • Callbacks — functions passed in to run later, can lead to deeply nested "callback hell"
  • Promises — objects representing a future value, with .then() / .catch()
  • async/await — syntax built on promises that lets asynchronous code read like synchronous code
async function getUser(){
  const res = await fetch('/api/user');
  const data = await res.json();
  return data;
}

Hoisting is JavaScript's behaviour of moving variable and function declarations to the top of their scope during compilation, before code runs. var declarations are hoisted and initialised as undefined; let and const are hoisted but stay in a “temporal dead zone” until their line runs, so accessing them earlier throws an error.

console.log(x); // undefined, not an error
var x = 5;

A closure is formed when a function “remembers” the variables from the scope it was created in, even after that outer function has finished running. Closures are the basis for private state and things like counters or memoized functions.

function counter(){
  let count = 0;
  return () => ++count;
}
const inc = counter();
inc(); // 1
inc(); // 2

undefined means a variable has been declared but has not yet been assigned a value, or a function returned nothing. null is an intentional assignment representing “no value”, explicitly set by the developer. typeof undefined is 'undefined', while typeof null is (confusingly) 'object', a long-standing quirk of the language.

Template literals use backticks instead of quotes and support embedded expressions with ${ ... } as well as multi-line strings, making string building far cleaner than concatenation.

const name = 'NM';
const msg = `Hello, ${name}!
Welcome to Full Stack.`;

Destructuring lets you unpack values from arrays or properties from objects into individual variables in one concise statement.

const [first, second] = [10, 20];
const { name, age } = { name: 'Aditi', age: 22 };

The same ... syntax does two related jobs. Spread expands an array or object into individual elements, useful for copying or merging. Rest collects multiple arguments or remaining items into a single array.

const merged = { ...obj1, ...obj2 }; // spread

function sum(...nums){                // rest
  return nums.reduce((a,b)=>a+b, 0);
}

  • map() — transforms every item and returns a new array of the same length
  • filter() — returns a new array containing only items that pass a test
  • reduce() — combines every item into a single accumulated value
const nums = [1,2,3,4];
nums.map(n => n*2);            // [2,4,6,8]
nums.filter(n => n % 2 === 0); // [2,4]
nums.reduce((sum,n)=>sum+n,0); // 10

JavaScript runs synchronous code first, top to bottom. Asynchronous work (timers, promises, network calls) is queued and only runs once the current synchronous code has finished, via the event loop — microtasks (promises) run before macrotasks (like setTimeout).

console.log('1');
setTimeout(()=>console.log('2'), 0);
Promise.resolve().then(()=>console.log('3'));
console.log('4');
// Output: 1, 4, 3, 2
