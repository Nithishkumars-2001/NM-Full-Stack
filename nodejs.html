Node.js

Node.js lets JavaScript run outside the browser, powering fast, scalable backend servers and APIs. This file covers the essentials for backend development.

Node.js is a runtime environment, built on Chrome's V8 engine, that lets JavaScript run outside the browser — on a server or a local machine. Unlike browser JavaScript, Node has no window or document, but it gains access to the file system, networking, and operating system through built-in modules.

npm (Node Package Manager) installs and manages third-party libraries. package.json is the manifest file for a Node project — it lists dependencies, scripts, and project metadata. 

npm init -y
npm install express
npm run start

Node.js runs JavaScript on a single thread, but heavy operations like file access or network requests are handed off to the system and completed asynchronously. The event loop constantly checks whether any of those operations have finished and runs their callback when ready, which is why Node can handle many requests at once without waiting on any single one.

Express is a minimal, flexible web framework for Node.js that simplifies building servers and APIs — handling routing, requests and responses without writing raw HTTP handling code from scratch.

const express = require('express');
const app = express();

app.listen(3000, () => console.log('Server running'));

A REST API exposes routes that respond to HTTP methods (GET, POST, PUT, DELETE) and typically return JSON. 

app.get('/api/users', (req, res) => {
  res.json([{ id: 1, name: 'Aditi' }]);
});

app.post('/api/users', (req, res) => {
  const newUser = req.body;
  res.status(201).json(newUser);
});

Middleware are functions that run between receiving a request and sending a response — used for logging, authentication, parsing request bodies, and error handling. Each middleware can inspect or modify the request before passing control on with next().

app.use(express.json());

app.use((req, res, next) => {
  console.log(req.method, req.url);
  next();
});

Node connects to MySQL using a driver package such as mysql2, creating a connection (or pool) and running SQL queries from JavaScript.

const mysql = require('mysql2');
const db = mysql.createConnection({
  host: 'localhost', user: 'root',
  password: '', database: 'nm_fullstack'
});

db.query('SELECT * FROM users', (err, rows) => {
  console.log(rows);
});

require() is Node's original CommonJS module system, loaded synchronously. import/export is the newer ES Modules standard (the same syntax used in modern frontend JavaScript), which Node also supports when a file uses the .mjs extension or "type": "module" is set in package.json. 

// CommonJS
const express = require('express');
module.exports = router;

// ES Modules
import express from 'express';
export default router;

npm installs and manages packages. npx executes a package's binary directly, downloading it temporarily if it isn't already installed, which is handy for one-off commands or project scaffolding tools without installing them globally.

npm install express        # install as a dependency
npx create-react-app app   # run a tool once, no permanent install

Environment variables store configuration (API keys, database credentials, ports) outside of the source code, so sensitive values aren't hard-coded or committed to version control. They're commonly loaded from a .env file using the dotenv package.

require('dotenv').config();
const dbPassword = process.env.DB_PASSWORD;

  • GET — retrieve data
  • POST — create a new resource
  • PUT — replace a resource entirely
  • PATCH — partially update a resource
  • DELETE — remove a resource

Express supports a special error-handling middleware with four parameters, placed after all other routes, which catches errors passed via next(err) or thrown in async code (with proper handling).

app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({ error: 'Something went wrong' });
});

CORS (Cross-Origin Resource Sharing) is a browser security feature that blocks web pages from making requests to a different domain than the one that served them, unless the server explicitly allows it via response headers. Node APIs commonly use the cors package to enable this for a frontend hosted on a different origin.

const cors = require('cors');
app.use(cors());

A single connection opens one channel to the database, which can become a bottleneck under load. A connection pool maintains a set of reusable connections that requests borrow and return, handling many simultaneous queries far more efficiently — the standard choice for production APIs.

const pool = mysql.createPool({
  host: 'localhost', user: 'root', database: 'nm_fullstack',
  connectionLimit: 10
});

JWT (JSON Web Token) is a compact, signed token used to verify a user's identity on each request without the server storing session state. After login, the server issues a token; the client sends it back (usually in an Authorization header) on subsequent requests, and the server verifies its signature before granting access.

const token = jwt.sign({ userId: user.id }, secretKey, { expiresIn: '1h' });

The fs module offers both. Synchronous methods (e.g. fs.readFileSync) block the event loop until the operation completes. Asynchronous methods (e.g. fs.readFile with a callback, or fs.promises.readFile with async/await) let Node continue handling other requests while waiting, which is why they're preferred in server code.

const data = await fs.promises.readFile('data.json', 'utf-8');
