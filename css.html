CSS

Cascading Style Sheets control colour, layout, spacing and responsiveness. This file walks through the concepts that come up in almost every project.

CSS (Cascading Style Sheets) describes how HTML elements should look — colours, fonts, spacing, layout and animation. It can be linked to a page in three ways:

<!-- External (recommended) -->
<link rel="stylesheet" href="style.css">

<!-- Internal -->
<style>body{ color:#333; }</style>

<!-- Inline -->
<p style="color:red;">Text</p>

Every element is treated as a rectangular box made up of four layers, from the inside out:

  • Content — the actual text or media
  • Padding — space between content and border
  • Border — the edge of the box
  • Margin — space outside the border, separating it from other elements

Setting box-sizing: border-box; makes padding and border count inside the declared width, which is why most modern stylesheets apply it globally.

A class selector (.card) can match many elements; an id selector (#header) should match only one. In terms of specificity — the rule CSS uses to decide which style wins when several rules target the same element — id selectors are stronger than class selectors, which are stronger than tag selectors like div. 

#header{ color:blue; }   /* wins over */
.card{ color:red; }      /* which wins over */
div{ color:green; }

  • relative — positioned relative to its own normal position
  • absolute — removed from normal flow, positioned relative to the nearest positioned ancestor
  • fixed — positioned relative to the browser window and stays put when scrolling
  • sticky — behaves like relative until a scroll threshold is reached, then behaves like fixed

Flexbox is a one-dimensional layout system for arranging items in a row or a column, with easy control over alignment, spacing and order. It's ideal for navbars, card rows, centering content, and any layout that flows in a single direction.

.container{
  display:flex;
  justify-content:space-between;
  align-items:center;
  gap:1rem;
}

CSS Grid is a two-dimensional layout system that controls rows and columns at the same time, making it ideal for full page layouts and dashboards. Flexbox is best for one-dimensional arrangements (a single row or column); Grid is best when you need precise control over both axes together.

.grid{
  display:grid;
  grid-template-columns: repeat(3, 1fr);
  gap:1.5rem;
}

Media queries let you apply different CSS rules depending on the characteristics of the device — most commonly screen width — so a page can adapt to phones, tablets and desktops. This is the foundation of responsive design.

@media (max-width: 768px){
  .nav{ flex-direction: column; }
}

  • px — a fixed pixel size, doesn't scale with anything
  • em — relative to the font size of the parent element (can compound)
  • rem — relative to the font size of the root (html) element, which avoids compounding issues
  • % — relative to the parent's corresponding property, e.g. width

rem is generally preferred for consistent, predictable spacing and typography.

Inline CSS is written directly on an element via the style attribute and has the highest specificity but is hard to maintain. Internal CSS lives in a <style> block in the document head, scoped to that one page. External CSS lives in a separate .css file linked with <link>, and is the standard approach since it can be cached and reused across pages.

A pseudo-class (single colon, :hover, :focus, :nth-child()) targets an element in a particular state. A pseudo-element (double colon, ::before, ::after, ::first-line) targets a specific part of an element or inserts generated content.

a:hover{ color: teal; }
p::first-line{ font-weight: bold; }

The cascade decides which rule applies when several rules could match the same element, based on origin, specificity, and source order (later rules win ties). Inheritance means some properties, mostly text-related ones like color and font-family, are automatically passed down from parent to child unless overridden. Layout properties like margin and border are not inherited by default.

display: none removes the element from the layout entirely — it takes up no space and is skipped by screen readers. visibility: hidden hides the element visually but it still occupies its space in the layout.

Custom properties let you define reusable values with --name syntax and read them with var(). Unlike preprocessor variables, they are live in the browser and can be changed at runtime, including per-component via cascading.

:root{ --brand-color: #8b5cf6; }
.btn{ background: var(--brand-color); }

A transition smoothly interpolates a property between two states, usually triggered by a state change like :hover. An animation uses @keyframes to define multiple steps and can run automatically, repeat, and control timing at each stage without needing a trigger.

.box{ transition: transform .3s ease; }
.box:hover{ transform: scale(1.05); }

@keyframes pulse{ 50%{ opacity:.5; } }
.dot{ animation: pulse 1.2s infinite; }

width sets a fixed target size. max-width lets an element shrink below that size on smaller screens but never grow past it — commonly used for responsive images and containers. min-width prevents an element from shrinking below a certain size, which can be useful for buttons or form fields that shouldn't get too small to use.

Specificity is a weight the browser calculates for each selector to resolve conflicts. It's typically expressed as four parts (inline, IDs, classes/attributes/pseudo-classes, elements/pseudo-elements) — more of a higher-value part always beats more of a lower one. 

#nav .link:hover  → 0-1-1-1 (1 id, 1 class-level, 1 pseudo)
.link              → 0-0-1-0
a                   → 0-0-0-1

!important overrides normal specificity entirely and should be used sparingly.
