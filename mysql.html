MySQL is a popular open-source relational database management system (RDBMS). A relational database stores data in structured tables made up of rows and columns, and relationships between tables are defined using keys, which keeps data organised and avoids duplication.

  • Database — a container that holds related tables
  • Table — a structured collection of data, similar to a spreadsheet
  • Row (record) — a single entry in a table
  • Column (field) — a specific attribute shared by every row, with a defined data type

A primary key uniquely identifies each row in a table and cannot be null or duplicated. A foreign key is a column in one table that references the primary key of another table, creating a relationship between them.

CREATE TABLE orders (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

SELECT name, email FROM users WHERE id = 1;

INSERT INTO users (name, email)
VALUES ('Aditi', 'aditi@example.com');

UPDATE users SET email = 'new@example.com' WHERE id = 1;

DELETE FROM users WHERE id = 1;

These four statements — often remembered as CRUD (Create, Read, Update, Delete) — cover almost everything you'll do with data day to day.

A JOIN combines rows from two or more tables based on a related column, usually a foreign key.

  • INNER JOIN — only matching rows in both tables
  • LEFT JOIN — all rows from the left table, matched rows from the right (or NULL)
  • RIGHT JOIN — the mirror of LEFT JOIN
  • FULL JOIN — all rows from both, matched where possible
SELECT orders.id, users.name
FROM orders
INNER JOIN users ON orders.user_id = users.id;

Normalization is the process of organising tables to reduce data duplication and improve data integrity, usually by splitting large tables into smaller, related ones. It's commonly done in stages called normal forms (1NF, 2NF, 3NF), each removing a specific kind of redundancy.

WHERE filters individual rows before any grouping happens. HAVING filters groups after a GROUP BY has been applied, and is typically used with aggregate functions like COUNT() or SUM(). 

SELECT department, COUNT(*) AS staff
FROM employees
WHERE active = 1
GROUP BY department
HAVING COUNT(*) > 5;

An index is a data structure that allows the database to look up rows much faster than scanning the entire table — similar to an index at the back of a book. Indexes speed up SELECT queries and joins significantly, but they add some overhead to INSERT/UPDATE operations, so they should be added deliberately on columns that are searched or joined often. 

CREATE INDEX idx_users_email ON users(email);

  • One-to-one — one row in table A relates to exactly one row in table B
  • One-to-many — one row in table A relates to many rows in table B (most common, e.g. one user has many orders)
  • Many-to-many — rows in both tables can relate to many rows in the other, usually implemented with a junction table

  • DELETE — removes specific rows (can use WHERE), can be rolled back, keeps the table structure
  • TRUNCATE — removes all rows quickly, resets auto-increment, cannot target specific rows
  • DROP — removes the entire table structure and its data permanently

Aggregate functions perform a calculation across multiple rows and return a single value. Common ones include COUNT(), SUM(), AVG(), MIN(), and MAX(), usually combined with GROUP BY.

SELECT department, AVG(salary) AS avg_salary
FROM employees
GROUP BY department;

A subquery is a query nested inside another query, used to filter or compute a value based on the result of the inner query first.

SELECT name FROM users
WHERE id IN (
  SELECT user_id FROM orders WHERE total > 100
);

Both enforce uniqueness on a column, but a table can have only one PRIMARY KEY (which also cannot be NULL), while it can have multiple UNIQUE constraints, and unique columns can typically accept one NULL value depending on the database.

A transaction groups multiple SQL statements so they either all succeed or all fail together, using START TRANSACTION, COMMIT, and ROLLBACK. ACID describes the guarantees a reliable transaction should have: Atomicity, Consistency, Isolation, Durability.

START TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;

A table physically stores data. A view is a saved, virtual query — it doesn't store data itself but presents the result of a query as if it were a table, which is useful for simplifying complex or frequently reused queries.

CREATE VIEW active_users AS
SELECT id, name FROM users WHERE active = 1;

ORDER BY sorts results, ascending (ASC, default) or descending (DESC). LIMIT restricts how many rows are returned, often used together for things like “top 5 highest-paid employees” or pagination.

SELECT name, salary FROM employees
ORDER BY salary DESC
LIMIT 5;
