HTML (HyperText Markup Language) is the standard markup language used to create the structure and content of web pages. It describes the different parts of a page — headings, paragraphs, links, images, forms — using a system of elements and tags, which browsers then read and render visually.

HTML by itself only handles structure; visual styling is handled by CSS and behaviour is handled by JavaScript.

A tag is the markup itself, written inside angle brackets, e.g. <p> (opening tag) and </p> (closing tag). An element is the complete unit: the opening tag, the content inside it, and the closing tag together. 

<p>This is a paragraph element.</p>

Some elements, called void elements, never have a closing tag or content, e.g. <img> and <br>.

Semantic elements clearly describe their meaning to both the browser and the developer, instead of being a generic container. They make a page more accessible and easier for search engines to understand.

<header>, <nav>, <main>, <section>,
<article>, <aside>, <footer>, <figure>

Compare this to a non-semantic tag like <div>, which carries no meaning on its own and is used purely as a generic wrapper.

Block-level elements always start on a new line and take up the full width available, e.g. <div>, <p>, <h1>–<h6>, <section>.

Inline elements only take up as much width as their content and sit within a line of text, e.g. <span>, <a>, <strong>, <img>. CSS can override this default behaviour using the display property.

<!DOCTYPE html> is not an HTML tag — it's an instruction to the browser that tells it to render the page using the modern HTML5 standard, rather than falling back to an older “quirks mode” that can cause inconsistent layout and spacing across browsers.

A hyperlink is created with the anchor tag and an href attribute pointing to the destination:

<a href="https://example.com" target="_blank">Visit site</a>

An image is inserted with the self-closing <img> tag, which requires a src and should always include descriptive alt text for accessibility: 

<img src="logo.png" alt="Company logo">

class can be applied to many elements on a page and is used for styling groups of elements the same way, or selecting them together in CSS/JavaScript.

id must be unique within the page and is typically used to target one specific element, for example as a jump-link anchor or a JavaScript hook.

<div class="card" id="pricing-card">...</div>

The <form> element collects user input and sends it to a server for processing, using attributes like action (where to send it) and method (GET or POST).

Common input types include:

  • text, email, password
  • number, date, checkbox, radio
  • <select> for dropdowns and <textarea> for multi-line text
  • submit to send the form

An attribute is defined in the HTML markup itself and sets the initial value, e.g. <input value="hi">. A property is the current value of that attribute on the live DOM object, accessed in JavaScript, e.g. input.value. They usually start in sync, but once the user types into the input, the property changes while the original attribute stays the same.

Void elements never contain content and never have a closing tag, because they represent a single piece of markup rather than a container. Common examples include <br>, <hr>, <img>, <input>, <meta> and <link>.

<div> is a block-level generic container used to group larger sections of content. <span> is an inline generic container used to wrap a small piece of text or an element within a line, typically for styling or scripting hooks.

<p>This is <span class="highlight">important</span>.</p>

<meta> tags provide metadata about a page that isn't displayed directly but is used by browsers and search engines — character encoding, page description, author, and the viewport setting that makes a page responsive.

<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<meta name="description" content="Page summary for search engines">

Both let a page store key-value data directly in the browser, with no server involved.

  • localStorage — persists even after the browser is closed, until explicitly cleared
  • sessionStorage — cleared automatically when the browser tab is closed
localStorage.setItem('theme', 'dark');
const theme = localStorage.getItem('theme');

Setting max-width: 100%; height: auto; on images keeps them from overflowing their container. The <picture> element and the srcset attribute let the browser choose an appropriately sized image for the device.

<img src="photo.jpg" srcset="photo-480.jpg 480w, photo-1080.jpg 1080w"
     sizes="(max-width: 600px) 480px, 1080px" alt="Landscape">

HTML entities represent characters that would otherwise be interpreted as markup, or characters not easily typed. They start with & and end with ;.

&lt;  → <
&gt;  → >
&amp; → &
&nbsp; → non-breaking space

Use semantic elements instead of generic <div>s wherever possible, always provide meaningful alt text on images, label form inputs with <label>, ensure sufficient color contrast, and make sure interactive elements are reachable and usable with a keyboard alone. ARIA attributes like aria-label can fill gaps where semantics aren't enough.
