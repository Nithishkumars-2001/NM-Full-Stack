React

React is a JavaScript library for building fast, component-based user interfaces. This file introduces the concepts you need to build a real React app.

React is a JavaScript library, created by Meta, for building user interfaces out of small, reusable components. Instead of manually updating the DOM, developers describe what the UI should look like for a given state, and React efficiently updates the page to match.

JSX is a syntax extension that lets you write HTML-like markup directly inside JavaScript. It gets compiled into regular React.createElement() calls behind the scenes.

function Welcome(){
  return <h1>Hello, NM Full Stack!</h1>;
}

A component is a reusable, self-contained piece of UI, written as a function (or class) that returns JSX. Props (properties) are read-only inputs passed into a component from its parent, similar to function arguments, letting the same component render differently based on the data it receives.

function Greeting({ name }){
  return <p>Hello, {name}!</p>;
}

<Greeting name="Aditi" />

State is data a component manages internally and can change over time, usually in response to user interaction. Unlike props, which are passed in from outside and read-only, state is owned and updated by the component itself using the useState hook.

const [count, setCount] = useState(0);

Hooks are functions that let function components use React features that used to require class components.

  • useState — adds local state to a component
  • useEffect — runs side effects (data fetching, subscriptions, timers) after render, and can clean up after itself
useEffect(() => {
  fetch('/api/data').then(r => r.json()).then(setData);
}, []); // empty array = run once on mount

The virtual DOM is a lightweight, in-memory copy of the real DOM. When state changes, React builds a new virtual DOM tree, compares ("diffs") it against the previous one, and updates only the parts of the real DOM that actually changed — which is far faster than re-rendering everything.

Conditional rendering typically uses standard JavaScript operators inside JSX, such as ternaries or &&. Lists are rendered with .map(), and each item needs a unique key prop so React can track it efficiently.

{isLoggedIn ? <Dashboard /> : <Login />}

{users.map(u => <li key={u.id}>{u.name}</li>)}

Class components had lifecycle methods like componentDidMount, componentDidUpdate and componentWillUnmount. In function components, useEffect covers all three: code in the effect body runs after mount/update, and a returned function runs as cleanup before unmount or before the effect re-runs.

In a controlled component, form input values are driven entirely by React state — every keystroke updates state via onChange. In an uncontrolled component, the DOM itself keeps track of the value, and React reads it on demand using a ref. Controlled components are the more common, predictable pattern.

const [value, setValue] = useState('');
<input value={value} onChange={e => setValue(e.target.value)} />

Prop drilling happens when data has to be passed down through several layers of components that don't need it themselves, just to reach a deeply nested child. It can be avoided using the Context API, or a state management library, to make data available directly to any component that needs it.

Context lets you share values (theme, logged-in user, language) across a component tree without manually passing props at every level. A Provider supplies the value, and any descendant can read it with useContext.

const ThemeContext = createContext('light');

function Toolbar(){
  const theme = useContext(ThemeContext);
  return <div className={theme}>...</div>;
}

useRef creates a mutable value that persists across renders without triggering a re-render when it changes. It's most commonly used to directly reference a DOM element, e.g. to focus an input, or to store a value like a timer ID between renders.

const inputRef = useRef(null);
<input ref={inputRef} />
<button onClick={() => inputRef.current.focus()}>Focus</button>

A key is a stable, unique identifier given to each item rendered from a list. React uses keys to tell items apart between renders, so it can update, reorder or remove exactly the right DOM nodes instead of re-rendering the whole list. Using array index as a key is discouraged if the list can be reordered or filtered.

React Router is the standard library for client-side routing in React — rendering different components based on the URL without a full page reload, which is how single-page applications get multiple “pages”.

<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/about" element={<About />} />
</Routes>

Data fetching is typically done inside useEffect so it runs after the component mounts, with the result stored in state so the UI re-renders when it arrives.

const [users, setUsers] = useState([]);

useEffect(() => {
  fetch('/api/users')
    .then(res => res.json())
    .then(setUsers);
}, []);

A traditional multi-page site requests a fresh HTML page from the server on every navigation. A single-page application, like a typical React app, loads once and then updates the page dynamically with JavaScript as the user navigates, only fetching data (not full HTML) from the server — giving a faster, app-like feel.
